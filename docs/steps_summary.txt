Edge-Enhancing Bi-Histogram Equalisation using Guided Image Filter
Week 1 -- Basic Steps Across All Modules
=======================================================================
(Refer to docs/data_metrics.txt, src/member1/README.md,
         src/member2/README.md, src/member3/README.md for full detail)
=======================================================================


MODULE 1 -- Data Ingestion, Colour-Space Extraction & Evaluation Metrics
-------------------------------------------------------------------------
Source: src/member1/data_metrics.h

  Step 1.  Load the input image using cv2.IMREAD_UNCHANGED to preserve
           native bit depth and channel count.

  Step 2.  Detect whether the image is grayscale (2-D) or colour (3-D).
           Scale 16-bit images down to 8-bit before processing.

  Step 3.  Extract the V (Value / brightness) channel.
             - Grayscale input  : copy as-is.
             - Colour input     : V = max(R, G, B) per pixel.
           The H and S channels are kept aside for Week 2 reconstruction.

  Step 4.  Compute five baseline quality metrics on the V channel:
             - AMBE  : mean pixel intensity (brightness baseline)
             - STD   : standard deviation   (global contrast)
             - CII   : contrast improvement index (local 3x3 contrast)
             - DE    : detail enhancement   (average edge sharpness)
             - PSI   : peak signal index    (highlight clipping ratio)
           These values are stored; Week 2 will re-compute them on the
           enhanced image to measure objective improvement.


MODULE 2 -- Histogram Statistics & Adaptive Plateau Clipping
-------------------------------------------------------------
Source: src/member2/statistics.h

  Step 1.  Build a 256-bin integer histogram from the V channel.
           Each bin h[k] counts pixels with intensity exactly k.

  Step 2.  Compute the PDF by dividing each bin by total pixel count N.
           All 256 values sum to 1.0.

  Step 3.  Compute the CDF as the running sum of the PDF.
           CDF[k] ranges from 0 to 1 and is non-decreasing.

  Step 4.  Segment the histogram at the mean intensity I_m (bi-HE split).
             - Lower sub-histogram : bins 0 .. I_m   (dark pixels)
             - Upper sub-histogram : bins I_m+1 .. 255 (bright pixels)
           Splitting at the mean prevents the brightness shift that
           standard whole-histogram equalisation causes.

  Step 5.  Compute the Adaptive Plateau Limit (APL) for each sub-histogram
           using its Shannon entropy:
             T    = N_sub / n_bins          (mean bin frequency)
             E    = -sum( p_i * log2(p_i) ) (sub-histogram entropy)
             Emax = log2(n_bins)            (maximum possible entropy)
             PL   = max(1, T * E / Emax)
           Clip any bin exceeding PL, then rebuild a normalised CDF for
           each sub-histogram. The result is a 256-element modified CDF
           ready for pixel remapping in Week 2.


MODULE 3 -- Guided Filter Coefficients & Visualisation
------------------------------------------------------
Source: src/member3/guided_filter.h  |  python/member3/visualize.py

  Step 1.  For every pixel in the V channel, scan its 3x3 neighbourhood
           and compute local statistics:
             mu_k    = mean of the 9 window pixels
             sigma2_k = variance of the 9 window pixels

  Step 2.  Compute the two Guided Filter coefficients per pixel:
             a_k = sigma2_k / (sigma2_k + epsilon)   where epsilon = 0.01 * 255^2
             b_k = mu_k * (1 - a_k)
           a_k near 1.0 means an edge/texture pixel -- preserve it.
           a_k near 0.0 means a flat/smooth pixel -- equalise it fully.
           The outer loop is parallelised with OpenMP for speed.

  Step 3.  Return a_coeff and b_coeff as H x W float64 arrays.
           In Week 2 these are applied as:
             V'_i = a_i * CDF_mapped(V_i) + b_i

  Step 4.  Generate and save two visualisation figures (PNG):
             week1_gf_visual.png
               - Original colour image
               - V channel (grayscale brightness map)
               - a-coefficient heatmap  (edge gain per pixel)
               - b-coefficient map      (brightness offset per pixel)
               - Edge pixels highlighted in red (a > 0.5)
               - Colour vs V channel side-by-side comparison

             week1_distribution_visual.png
               - Full 256-bin pixel intensity histogram
               - Bi-histogram split at mean I_m
               - PDF (bars) overlaid with CDF (line)
               - Original CDF vs APL-clipped bi-CDF comparison


=======================================================================
OVERALL PIPELINE SEQUENCE
=======================================================================

  Image
    |
    v
  [Module 1]  extract V channel, compute 5 baseline metrics
    |
    v
  [Module 2]  histogram -> PDF -> CDF -> bi-split -> APL clip -> modified CDF
    |
    v
  [Module 3]  GF coefficients (a, b) per pixel  +  save visualisations
    |
    v
  Week 2:  V'_i = a_i * CDF_mapped(V_i) + b_i
           Reconstruct colour image with enhanced V + original H, S
           Re-compute 5 metrics -> compare vs baseline

Data Ingestion, Colour-Space Extraction & Evaluation Metrics
=======================================================================

OVERVIEW
--------
This module is responsible for the very first stage of the pipeline:
reading the raw image, converting it into a form the algorithm can work
with, and measuring the quality of any image using five standard metrics.
These metrics act as the baseline -- they are computed on the original
image in Week 1, and again on the enhanced image in Week 2 to objectively
show how much the algorithm improved the picture.

All code is written in C++17 using raw pointer arithmetic (no OpenCV in
the backend). Functions are exposed to Python via pybind11.

SOURCE FILE
-----------
  src/member1/data_metrics.h

=======================================================================
WHAT THIS MODULE IMPLEMENTS
=======================================================================

1. V-CHANNEL EXTRACTION  --  extract_v_channel
-----------------------------------------------------------------------
The paper operates on the Value (brightness) channel of an image, not
the full colour image. This function performs that extraction.

  - Grayscale image (2-D array): returns a plain copy -- the pixels
    already represent brightness.
  - Colour image (3-D, H x W x 3): computes V = max(R, G, B) for every
    pixel.

Using max(R, G, B) instead of a weighted grey conversion preserves peak
luminance faithfully, which is important for histogram equalisation to
work correctly. The Hue and Saturation channels are NOT discarded --
they are stored separately in Python and will be recombined with the
enhanced V channel in Week 2 to reconstruct a full colour output.

  Input : H x W (grayscale)  or  H x W x 3 (colour BGR)
  Output: H x W single-channel uint8 brightness map


2. MEAN BRIGHTNESS  --  compute_ambe
-----------------------------------------------------------------------
Computes the mean pixel intensity of the V channel:

  AMBE = (1/N) * sum( V_i )

In Week 2, the Absolute Mean Brightness Error is:

  AMBE = | mean(V_original) - mean(V_enhanced) |

A lower AMBE means the algorithm preserved the natural brightness of
the image rather than making it artificially too bright or too dark.


3. STANDARD DEVIATION  --  compute_std
-----------------------------------------------------------------------
Measures the global contrast of the image -- how spread out the pixel
intensities are:

  sigma = sqrt( (1/N) * sum( (V_i - mean(V))^2 ) )

A higher standard deviation after enhancement means the algorithm
successfully stretched the contrast and made the image more visually
distinct.


4. CONTRAST IMPROVEMENT INDEX  --  compute_cii
-----------------------------------------------------------------------
Measures local contrast using a 3x3 sliding window around each pixel:

  C_loc = (V_max - V_min) / (V_max + V_min)

The CII is the average of all local contrast values across the image.
It captures fine texture and edge contrast that global standard
deviation would miss.


5. DETAIL ENHANCEMENT  --  compute_de
-----------------------------------------------------------------------
Measures how well fine details (edges, textures) are preserved or
enhanced. Computes the average absolute difference between each pixel
and its 3x3 neighbourhood mean:

  DE = (1/N) * sum( |V_i - local_mean_i| )

A higher DE after enhancement means the algorithm sharpened fine
structural details in the image.


6. PEAK SIGNAL INDEX  --  compute_psi
-----------------------------------------------------------------------
Counts the proportion of pixels at or near the maximum intensity
(clipped highlights):

  PSI = (pixels where V >= 250) / N

A lower PSI is better -- it means the algorithm did not over-brighten
the image and clip highlight detail.
